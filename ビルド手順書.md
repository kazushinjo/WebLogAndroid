# WebLog for Android — ビルド手順書

---

## 必要なもの

| 項目 | 要件 |
|------|------|
| PC/Mac | Windows / macOS / Linux |
| Android Studio | Ladybug (2024.2.1) 以降推奨 |
| JDK | 17 以上（Android Studio 同梱） |
| Android 端末 | Android 8.0 (API 26) 以上 |

## 手順 1 — Android Studio をインストールする

1. [developer.android.com/studio](https://developer.android.com/studio) から Android Studio をダウンロード
2. インストール後、初回起動時の SDK セットアップを完了する
   - **SDK Platform:** Android 14 (API 35) を選択
   - **SDK Tools:** デフォルトのまま Next

---

## 手順 2 — プロジェクトを開く

1. Android Studio を起動
2. **Open** をクリック
3. `Desktop/WebLogAndroid/` フォルダを選択して **OK**
4. Gradle の同期が自動的に開始されます（初回は数分かかります）

---

## 手順 3 — ランチャーアイコンを設定する（任意）

1. プロジェクトナビゲータで `app/src/main/res` を右クリック
2. **New > Image Asset** を選択
3. アイコン画像を設定して **Finish**

---

## 手順 4 — 実機でビルドする

### Android 端末の開発者モードを有効にする

1. **設定 > デバイス情報 > ビルド番号** を 7回タップ
2. **設定 > 開発者向けオプション > USB デバッグ** を ON
3. USB ケーブルで PC/Mac に接続
4. 「USB デバッグを許可しますか？」ダイアログで **許可**

### ビルドして実行

1. Android Studio 上部のデバイス選択で接続した端末を選択
2. **▶ Run** ボタンをクリック（または **Shift + F10**）
3. アプリが端末にインストールされて起動すれば成功

---

## 手順 5 — エミュレータで確認する（実機なしの場合）

1. **Device Manager** から **Create Device** でエミュレータを作成
   - **Pixel 8** などのプロファイルを選択
   - System Image: **Android 14 (API 35)** を選択
2. エミュレータを起動して ▶ Run

## トラブルシューティング

### ❌ Gradle の同期に失敗する

**対処:**
- Android Studio メニュー → **File > Sync Project with Gradle Files**
- ネットワーク接続を確認（初回ダウンロードに時間がかかります）

### ❌ 「Could not resolve ...」エラー

**対処:** `gradle/wrapper/gradle-wrapper.properties` の `distributionUrl` が正しいか確認。
オフライン環境では Maven Local へのキャッシュが必要です。

### ❌ jcc.json が読み込まれない

**対処:** `app/src/main/assets/jcc.json` が存在するか確認。
